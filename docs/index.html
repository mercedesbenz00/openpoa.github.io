OpenPOA Blockchain Network
The Original Ethereum Side-Chain

OpenPOA is a Proof of Authority (PoA) publicly accessible blockchain based on Ethereum Virtual Machine; created and maintained by a consortium of Blockchain developers, to aide the Blockchain developer & user community.

Why should developers use OpenPOA Blockchains?

Speed
Consistent 5-second blocktimes for a superior developer experience

Resilience
PoA Consensus prevents attacks from hindering development & bad actors

Reliability
Known Independent Validators hosting secure & reliable, independent nodes

Cheap
Transactions on OpenPOA Blockchains consume less gas & are validated quicker

Infrastructure
Block explorer powered by POA's Blockscout client & load-balanced RPC nodes

Innovation
Cross-chain bridge helps to transfer assets from chain-to-chain

Deploy your own Full Node on OpenPOA or use our supplied RPC Endpoint below to connect your dapp, wallet or developer environment to OpenPOA's Core or Sokol Chains:

Full OpenEthereum Node

OpenPOA Sokol Chain Spec & bootnodes on Github

git clone -b sokol https://github.com/openpoa/poa-chain-spec.git

Start OpenEthereum w/ flags

openethereum --chain /repo/path/spec.json --reserved-peers /repo/path/bootnodes.txt

3. Develop!
Deploy your Contracts

Once you have a Connection Method and some POA Coins, you're ready to go!

The Ethereum development ecosystem is ever-growing and you can use many of the same tools that you're already familiar with.

OpenPOA uses Solidity for on-chain smart contracts. This opens up endless opportunities for its developers, community & users to migrate their existing habits onto OpenPOA Sokol Testnet & Core Mainnet.

Truffle

Truffle is the most popular development framework for Ethereum. You can connect to OpenPOA Chains using one of the following methods:

Use OpenEthereum full node and add the config to your project's truffle-config.js:
opSokol: {
  host: 'localhost',
  port: 8545,
  network_id: '77'
}
Use an HD Wallet Provider to connect to a remote RPC node such as the one provided on this page, in the 'Connect to OpenPOA' Section
